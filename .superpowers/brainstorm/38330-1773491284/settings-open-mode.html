Settings Window: How should it open?

This affects the macOS feel, implementation complexity, and whether the main chat stays visible.

A

Separate Electron BrowserWindow

Opens as a distinct macOS window — has its own traffic lights, can be moved/resized independently. Matches the screenshot exactly. Requires a second renderer entry point and IPC wiring to open it from the Settings button.

Pros

  • Authentic macOS feel
  • Main chat stays accessible behind it
  • Native window management (Mission Control, etc.)

Cons

  • More complex: separate HTML entry + main-process IPC
  • Shared state across windows needs careful handling
B

Full-screen Overlay in Main Window

A React route/modal that slides in over the chat interface inside the same BrowserWindow. No new window created — just a React component covering the main view.

Pros

  • Much simpler: no IPC, no extra entry point
  • Shared React state out of the box
  • Faster to build

Cons

  • Not a real macOS window — no traffic lights, no Move
  • Main chat hidden while settings is open
  • Doesn't match the screenshot aesthetic
